# 雅思听力阅读本地刷题程序 V1 分阶段执行文档

> **给后续开发者/代理的要求：** 实施本文档时，按阶段逐项完成。每个阶段必须达到验收标准后再进入下一阶段。若使用 Superpowers 执行，推荐使用 `superpowers:subagent-driven-development` 或 `superpowers:executing-plans`。

**目标：** 开发一个 Mac/Windows 都可使用的本地雅思听力阅读刷题程序，支持模考、练习、精听、精读、题库更新、频率表更新、历史记录、正确率统计、答案解析、成绩预测和百度云共享文件夹同步。

**总体架构：** 采用本地优先 Web 程序。前端使用 React/Vite，后端使用 Node/Fastify，本地数据库使用 SQLite。每台电脑各自保存主数据库，跨设备只同步可合并的 JSONL 历史日志，不直接同步 SQLite 数据库，降低百度云冲突导致的数据损坏风险。

**技术栈：** TypeScript、React、Vite、Fastify、SQLite、Drizzle ORM、Zod、Vitest、Playwright、jszip、cheerio、mammoth、pdfjs-dist。Web 版稳定后，再用 Tauri 打包 Mac `.dmg` 和 Windows `.exe`。

## 一、V1 范围

V1 是给个人使用的本地程序，不做公开平台。

V1 包含：

- 练习模式：自由选题、自由答题、可看解析、可精听精读。
- 模考模式：严格计时、严格听力播放规则、整套提交、真实成绩估算。
- 题库管理：导入听力 ZIP、阅读 PDF、频率表，支持后续更新。
- 频率分类：高频、次高频、非高频，随机组卷时高频优先。
- 精听：听力音频播放、A-B 循环、句子/片段循环、听写、原文对照。
- 精读：阅读文章、答案句高亮、定位句、同义替换、详细解析。
- 历史记录：每次练习和模考都保存。
- 正确率统计：按听力 P1-P4、阅读 P1-P3、题型、频率、错因统计。
- 预测分数：根据模考和练习历史预测真实考试分数范围。
- 百度云同步：通过 `/Users/musheng/Desktop/同步空间/IELTS-Sync` 同步做题记录。
- Mac/Windows 支持：先本地 Web 运行，后续打包桌面版。

V1 不包含：

- 用户登录。
- 多用户管理。
- 在线支付。
- 云服务器。
- 公开题库平台。
- 自动购买或抓取版权内容。

## 三、核心数据表

SQLite 中至少需要这些表：

- `devices`：设备 ID、设备名称、系统、首次使用时间。
- `sources`：导入来源、文件路径、校验值、导入状态、版本号。
- `passages`：阅读文章或听力 section 元信息。
- `questions`：题干、题型、所属 part、顺序、答题规则。
- `answer_keys`：正确答案、可接受答案、答案句、解析、同义替换、字数限制。
- `listening_audio`：音频路径、时长、校验值。
- `listening_cues`：听力句子或片段的开始时间、结束时间、文本。
- `frequency_entries`：频率表来源、题目名称、part、频率分类、难度。
- `attempts`：一次练习或模考记录。
- `attempt_answers`：每题作答、是否正确、耗时、是否标记 review。
- `mistake_labels`：错因标签。
- `stats_snapshots`：统计快照和预测分。
- `sync_events`：已同步事件，用于跨设备去重。

## 六、阶段 2：题库导入

**目标：** 能导入你现有的听力 ZIP、阅读 PDF 和频率表。

### 听力 ZIP 导入

支持路径示例：

```text
/Users/musheng/Desktop/IELTS/listening/IELTS Listening 虾滑/P1/高频/*.zip
/Users/musheng/Desktop/IELTS/listening/IELTS Listening 虾滑VIP/P4/次高频/*.zip
```

需要解析：

- ZIP 文件名。
- part：P1/P2/P3/P4。
- 频率：高频、次高频、非高频。
- HTML 文件。
- DOCX 文件。
- PDF 文件。
- `audio.mp3`。

导入策略：

- 能结构化解析的题目直接入库。
- 无法稳定解析的题目标记为 `needs_review`。
- 原始文件复制到 `data/assets/listening/<sourceId>/`。
- 用文件校验值去重。

### 阅读 PDF 导入

支持路径：

```text
/Users/musheng/Desktop/IELTS/reading/ReadingPractice/PDF
```

需要解析：

- 文件名。
- part：P1/P2/P3。
- 英文题名。
- 中文题名。
- PDF 文件路径。

导入策略：

- 先保存文章资产和元信息。
- 如果 PDF 无法自动提取结构化题目，标记为 `needs_review`。
- 后续通过手动编辑补答案、解析、答案句。

### 频率表导入

支持：

- CSV。
- XLSX。
- 从图片 OCR 后人工校对。
- 你当前的 `reading/5月高频表格.jpg` 可作为频率表来源。

验收标准：

- 现有听力 ZIP 能批量出现在题库列表。
- 现有阅读 PDF 能批量出现在题库列表。
- 高频、次高频、非高频能正确进入数据库。

## 八、阶段 4：模拟考试系统

**目标：** 做出接近雅思机考的听力和阅读模考体验。

听力模考：

- 4 个 section。
- 共 40 题。
- 严格计时。
- 模考模式禁止暂停、拖动、倍速。
- 到时自动提交。
- 提交前提示未答题和标记题。

阅读模考：

- 3 篇文章。
- 共 40 题。
- 60 分钟倒计时。
- 左文章、右题目。
- 支持拖拽分栏宽度。
- 支持高亮和笔记。
- 到时自动提交。

通用考试界面：

- 顶部计时器。
- Help。
- Settings。
- 字体大小。
- 底部题号栏。
- Review 标记。
- End test / Submit。

验收标准：

- 听力和阅读都能完成完整模考。
- 时间结束自动提交。
- 生成 raw score 和 estimated band score。

## 九、阶段 5：频率优先随机组卷

**目标：** 根据频率表随机组卷，并优先选择高频题。

权重：

- 高频：5。
- 次高频：3。
- 非高频：1。
- 无频率数据：1。

近期重复降权：

- 最近 7 天做过：乘以 0.1。
- 最近 30 天做过：乘以 0.4。
- 从未做过：乘以 1。

组卷规则：

- 听力完整套题：P1、P2、P3、P4 各一篇。
- 阅读完整套题：P1、P2、P3 各一篇。
- 专项练习可按 part、题型、频率、错因筛选。

验收标准：

- 随机套题结构正确。
- 高频题出现概率明显更高。
- 最近刚做过的题不频繁重复。

## 十、阶段 6：精听和精读

**目标：** 让程序不只是测分，也能帮你提分。

### 精听

功能：

- 播放/暂停。
- 拖动进度。
- 倍速。
- A-B 循环。
- 单句或片段循环。
- 听写输入。
- 显示/隐藏原文。
- 保存听写记录。

关键设计：

- 单句循环依赖 `listening_cues`。
- 如果题包没有句子时间轴，先支持 A-B 循环。
- 提供 cue 编辑器，手动标记句子开始和结束时间。
- 有 cue 后即可使用单句反复听。

### 精读

功能：

- 阅读文章分栏。
- 题目与文章联动。
- 答案句高亮。
- 定位句。
- 关键词。
- 同义替换。
- 详细解析。
- 错因标签。

错因标签：

- 听力：漏听、拼写、数字日期、地图定位、同义替换、干扰项、审题错误。
- 阅读：定位失败、同义替换、判断题陷阱、段落结构、词汇障碍、时间不足、选项干扰。

验收标准：

- 听力能基于 cue 进行单句循环。
- 没有 cue 时能用 A-B 循环并手动创建 cue。
- 阅读解析能高亮答案句。
- 错题能记录错因。

## 十二、阶段 8：百度云同步和备份

**目标：** Mac 和 Windows 之间同步做题记录。

默认同步目录：

```text
/Users/musheng/Desktop/同步空间/IELTS-Sync
```

同步原则：

- 不把 `ielts.db` 放进百度云。
- 每台设备保留自己的本地 SQLite。
- 同步目录只保存 JSONL 事件日志。
- 每条事件都有 `eventId`、`deviceId`、`createdAt`。
- 启动程序时自动导入新事件。
- 用户也可以手动点击同步。

同步内容：

- attempts：练习和模考记录。
- answers：每题答案。
- mistakes：错因。
- stats：统计快照。
- frequency：频率表更新。
- imports：题库导入记录。

冲突处理：

- 同一事件重复出现时去重。
- 不覆盖已提交的本地答案。
- 如果两台设备对同一次 attempt 产生不同答案，保留两份并在复盘页显示冲突。
- 手动备份文件存入 `data/backups`。

验收标准：

- Mac 上完成一次练习后，Windows 可以导入该记录。
- Windows 上完成一次练习后，Mac 可以导入该记录。
- 同步过程中不会直接修改或覆盖 SQLite 文件。

## 十七、主要风险和处理方式

| 风险 | 影响 | V1 处理方式 |
| --- | --- | --- |
| 听力 HTML/PDF/DOCX 格式不统一 | 自动解析可能失败 | 保留原始文件，导入状态标记为 `needs_review`，提供人工校对入口 |
| 阅读 PDF 缺少结构化答案位置 | 无法自动答案句高亮 | V1 支持手动选择答案句 |
| 听力音频没有句子时间轴 | 无法自动单句循环 | V1 提供 A-B 循环和 cue 编辑器 |
| 百度云生成冲突副本 | 历史记录可能重复 | 使用 eventId、deviceId 和 JSONL 去重 |
| 雅思真实分数线会浮动 | 分数可能显得过度确定 | 所有 band 都标注为 estimated |
| 完全复制官方界面可能有版权/品牌风险 | 视觉设计风险 | 做考试体验仿真，不复制官方 logo 和受保护素材 |

## 十八、近期最小可用版本

如果你临近考试，建议先做一个可用 MVP：

1. 本地启动。
2. 样例题做题。
3. 听力 ZIP 批量导入。
4. 阅读 PDF 批量导入。
5. 手动补答案和解析。
6. 练习模式。
7. 阅读模考。
8. 听力模考。
9. 历史记录。
10. 正确率统计。

MVP 完成后，你就可以开始实际备考。精听 cue、精读高亮、预测分数、百度云同步再逐步补齐到完整 V1。
